1. Préparer le nouveau compte AWS
Crée un nouveau compte AWS (ou connecte-toi à un compte existant avec des crédits).
Active la facturation et les services nécessaires (IAM, EKS, EC2, VPC, S3, etc.).
2. Configurer l’utilisateur et les permissions
Crée un utilisateur IAM avec les droits administrateur (ou un rôle avec les permissions nécessaires pour Terraform).
Récupère les clés d’accès (Access Key ID et Secret Access Key).
3. Configurer AWS CLI sur ta machine
Lance : aws configure
Renseigne les nouvelles clés d’accès, la région (ex : us-east-1), et le format de sortie (json).
4. Mettre à jour le provider Terraform
Dans main.tf, vérifie que le provider AWS utilise la région souhaitée :
provider "aws" {
  region = "us-east-1"
}
Si tu veux utiliser un profil spécifique, ajoute :
provider "aws" {
  region  = "us-east-1"
  profile = "nom_du_profil"
}
5. Réinitialiser l’état Terraform
Supprime ou sauvegarde les fichiers terraform.tfstate et .terraform/ dans le dossier infra.
Relance :
terraform init
terraform plan
terraform apply
Cela va provisionner toutes les ressources sur le nouveau compte AWS.
6. Vérifier et adapter les modules
Les modules VPC, EKS, Security Groups, etc. sont déjà prêts.
Si tu utilises des ressources spécifiques (ECR, SQS, S3, etc.), vérifie que les modules sont bien configurés pour le nouveau compte.
7. Déployer les images Docker
Recrée les repositories ECR sur le nouveau compte.
Push tes images Docker sur le nouveau ECR :
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <nouveau_ecr_url>
docker tag <image> <nouveau_ecr_url>/<repo>:latest
docker push <nouveau_ecr_url>/<repo>:latest

8. Déployer les manifests Kubernetes
Après création du cluster EKS, mets à jour le kubeconfig :
aws eks update-kubeconfig --region us-east-1 --name <nouveau_cluster>
Applique les manifests :
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/backend.yaml -n magicfit
kubectl apply -f k8s/frontend.yaml -n magicfit
kubectl apply -f k8s/magicmirror.yaml -n magicfit
kubectl apply -f k8s/db.yaml -n magicfit
9. Vérifier les ressources sur AWS Console
Vérifie que tout est bien créé (VPC, subnets, SG, EKS, node groups, ECR, etc.).
Vérifie que les pods démarrent correctement.
10. Mettre à jour les secrets et variables d’environnement
Mets à jour les secrets dans AWS Secrets Manager pour le nouveau compte.
Mets à jour les endpoints, URLs, et credentials dans tes fichiers .env et manifests K8s.
11. Tester l’application
Accède à tes services via les LoadBalancers ou Ingress.
Vérifie le fonctionnement du backend, frontend, DB, MagicMirror, etc.

En résumé :

Prépare le nouveau compte AWS.
Configure AWS CLI et Terraform pour ce compte.
Réinitialise l’état Terraform.
Recrée et push les images Docker.
Déploie les manifests K8s.
Mets à jour les secrets et variables.
Teste l’application.
Si tu veux un script ou une checklist détaillée pour chaque étape, demande-le !
